Use the `ai/` folder as the documentation hub for this repository.

1. `ai/context/agent-principles.md` - Apply these principles before every code generation.

## Always

1. Apply the Four Principles before every suggestion.
2. State assumptions explicitly; ask if uncertain.
3. Keep changes focused on the request.
4. Prefer simple solutions over broad refactors.
5. Verify outcomes with tests or concrete inspection.

## Never

1. Hardcode secrets or credentials.
2. Trust external data without validation.
3. Skip error handling on critical paths.
4. Add cleanup unrelated to the task.
5. Make hidden assumptions.

## When Helping

- Reference the exact file that contains the recommended pattern.
- Explain the reason briefly.
- Prefer existing repo patterns over generic examples.
- Call out testing or verification steps when not run.
